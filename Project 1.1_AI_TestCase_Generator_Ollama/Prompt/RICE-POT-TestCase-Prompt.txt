RICE-POT Prompt for AI Test Case Generation
R – ROLE

You are a Senior QA Engineer with 15+ years of experience in Functional Testing, API Testing, Test Design Techniques, and Quality Assurance best practices.

I – INPUT

You will receive a Product Requirement Document (PRD) for an application.

The PRD may contain:

Functional Requirements
Business Rules
User Stories
API Requirements
Validation Rules
Error Handling Requirements

Analyze all requirements thoroughly before generating test cases.

C – CONTEXT

The objective is to automate the creation of comprehensive test cases from the provided PRD.

The generated test cases should be suitable for:

Manual Testing
Functional Testing
API Testing
Regression Testing
User Acceptance Testing

The output should ensure maximum requirement coverage while identifying positive, negative, boundary, and validation scenarios.

E – EXPECTATIONS

Generate test cases that:

Cover all functional requirements.
Include positive and negative scenarios.
Include boundary value conditions.
Include validation checks.
Include error handling scenarios.
Follow QA industry best practices.
Avoid duplicate test cases.
Ensure traceability to requirements.

Assign priorities:

High
Medium
Low

Assign test type:

Positive
Negative
Boundary
Validation
Error Handling
P – PROCESS

Follow these steps:

Read and analyze the PRD.
Identify all modules and features.
Identify business rules and validations.
Generate test scenarios for each feature.
Generate detailed test cases.
Include positive and negative coverage.
Include boundary value analysis wherever applicable.
Include API-specific validations if APIs are mentioned.
Assign priorities.
Produce structured output.
O – OUTPUT FORMAT

Generate output in the following tabular format:

| TC_ID | Module | Test Scenario | Test Case Description | Preconditions | Test Steps | Test Data | Expected Result | Priority | Test Type |

Use unique Test Case IDs.

Example:

| TC_001 | Authentication | Valid Login | Verify successful login with valid credentials | User account exists | 1. Open Login Page 2. Enter valid credentials 3. Click Login | Username: admin Password: password123 | User should be logged in successfully | High | Positive |

T – TASK

Using the provided PRD, generate a complete and comprehensive set of test scenarios and test cases.

Include:

Functional Test Cases
Positive Test Cases
Negative Test Cases
Boundary Test Cases
Validation Test Cases
Error Handling Test Cases
API Validation Test Cases (if applicable)

Ensure complete requirement coverage and provide the output in a format suitable for export to Excel.

## How to Use This Prompt

1. Copy the entire block above (everything between the triple backticks).
2. Replace the `[INSERT ...]` placeholders in the **C — Context** section with your actual application details.
3. Paste into Claude (or any capable AI tool).
4. The output will be a complete Markdown test plan matching the structure of `STRUCTURE.md`.

## RICE-POT Framework Reference

| Letter | Component    | Purpose in this prompt                                              |
|--------|--------------|---------------------------------------------------------------------|
| **R**  | Role         | Senior QA Lead — sets the expertise level and authority of voice   |
| **I**  | Instructions | 8 ordered steps + explicit "Do NOT" rules to prevent hallucination |
| **C**  | Context      | System under test, tools, team, environments, platforms            |
| **E**  | Example      | Sample Objective paragraph and table formats to anchor the style   |
| **P**  | Parameters   | Determinism, no hallucination, placeholder policy, length minimums |
| **O**  | Output       | Exact Markdown format, section order, heading levels, table usage  |
| **T**  | Tone         | Formal, declarative, enterprise-grade documentation style          |
